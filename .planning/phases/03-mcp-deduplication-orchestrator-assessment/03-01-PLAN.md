Create `src/mcp/DiscussionRunner.ts` with a class that encapsulates the shared discussion lifecycle:

**DiscussionRunnerOptions interface:**
```typescript
export interface DiscussionRunnerOptions {
  task: string;
  config?: string;           // config path for ConfigCascade.resolve
  projectPath?: string;      // project context path
  personas?: string;         // persona string for PersonaSystem
  rounds?: number;           // default 4
  minRounds?: number;        // default 2
  dynamic?: boolean;         // default false
  selectorModel?: string;    // default DEFAULT_SELECTOR_MODEL
  judgeModel?: string;       // override judge model
  judgeInstructions?: string;
  timeout?: number;          // default 0 (no timeout)
  contextOptimization?: boolean;
  priorHistory?: Array<{ role: string; content: string; speaker?: string }>;  // for continuation
  onProgress?: (event: { type: string; message: string }) => void;  // progress callback
  validateProjectPath?: (path: string) => string;  // path validator (REST has different validation)
}
```

**DiscussionResult interface:**
```typescript
export interface DiscussionResult {
  result: any;             // raw ConversationManager result
  sessionId: string;       // saved session ID
  logFilePath: string;     // saved discussion log path
  timedOut: boolean;
  effectiveTimeout: number;
}
```

**DiscussionRunner class with `async run(options: DiscussionRunnerOptions): Promise<DiscussionResult>`:**

The run() method consolidates the shared logic from all three handlers:
1. Config resolution: `ConfigCascade.resolve({ config: options.config })`
2. Context optimization: set `config.contextOptimization = { enabled: true }` if option set
3. Persona resolution: `PersonaSystem.getPersonas()` + `personasToAgents()` to set `config.agents`
4. Round validation: `config.max_rounds = rounds`, validate `min_rounds <= rounds`
5. Project context loading: `new ProjectContext(path)` + `await projectContext.load()` (with symlink check)
6. Judge creation: `{ model, provider: ProviderFactory.createProvider(model), systemPrompt }`
7. EventBus setup: `EventBus.createInstance()` + register named handlers for round:start, agent:thinking, error, status — forward to `onProgress` callback if provided
8. ConversationManager construction: `new ConversationManager(config, null, false, scopedEventBus, dynamic, selectorModel, { judgeInstructions })`
9. Timeout/abort: AbortController with effectiveTimeout = `timeout === 0 ? 0 : Math.max(timeout, 600)`
10. Prior history injection: if `priorHistory` provided, push each entry to `conversationManager.conversationHistory`
11. Progress heartbeat: `setInterval` every 30s forwarding to onProgress
12. Execute: `await conversationManager.startConversation(task, judge, projectContext)`
13. Cleanup in finally: clearInterval heartbeat, clearTimeout, remove EventBus listeners
14. Session saving: `SessionManager.createSessionManifest()` + `saveSession()`
15. Discussion log saving: `saveFullDiscussion(result)` — import from server.ts (export it)
16. Return `{ result, sessionId, logFilePath, timedOut, effectiveTimeout }`

The `saveFullDiscussion` function must be exported from server.ts (or moved to a utility). For minimal diff in this plan, export it from server.ts. Plan 02 will use the import.

Create `src/mcp/__tests__/DiscussionRunner.test.ts` with tests per the behavior spec above. Mock ConversationManager, EventBus, SessionManager, ConfigCascade, PersonaSystem, ProviderFactory.
