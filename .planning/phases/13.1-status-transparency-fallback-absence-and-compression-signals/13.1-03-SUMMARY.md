# Phase 13.1 Plan 03: TurnDistributionReporter cap-exclusion + participation + absence events Summary

Wired SpeakerSelector's existing max_turn_ratio cap into TurnDistributionReporter so Phase 13 fairness exclusions become first-class participation data, and added the end-of-discussion `conversation:agent_absent` emission downstream plans consume to populate the Run Integrity Participation subsection.

## What Was Built

### TurnDistributionReporter (src/core/TurnDistributionReporter.ts)

Extended the Phase 13 reporter — previously a stateless observer that only emitted `turn_distribution_updated` / `fairness_alarm` — with three new methods and the backing state they need:

- `recordCapExclusion(agent, round, ratio)` — dedup-per-(agent, round), stores first-exclusion ratio.
- `buildParticipationReport(configuredAgents, failedAgents)` — returns a `ParticipationEntry[]` covering every configured agent, with status resolved via (spoken | absent-failed | absent-capped | absent-silent) precedence.
- `finalizeAbsenceEvents(report)` — emits one `conversation:agent_absent` per non-spoken entry via the reporter's EventBus, guarded by an idempotency flag.

`report()` now updates an internal `_turnCounts` map from its snapshots so `buildParticipationReport()` can classify agents after the run without needing a separate turn-count source. The existing per-snapshot `turn_distribution_updated` / `fairness_alarm` emissions are unchanged (verified by regression test).

Constructor gained an optional `eventBus?: EventBus` parameter — used only by `finalizeAbsenceEvents`. All existing callers (`new TurnDistributionReporter()` with no args) continue to work; they just won't be able to finalize absence events until wired in a later plan.

### SpeakerSelector (src/core/SpeakerSelector.ts)

`selectNextSpeaker` gained an optional trailing `turnDistributionReporter?: TurnDistributionReporter` parameter. Inside the existing Phase 13 `max_turn_ratio` cap branch, after filtering dropped agents and before replacing `candidates`, the selector computes `observedRatio = droppedTurns / mean` for each dropped agent and calls `recordCapExclusion(agent, currentRound, observedRatio)`.

Fairness cap behavior is unchanged — this is pure observation, not mutation. Existing SpeakerSelector + SpeakerSelector.fairness tests still pass (41 tests green across both files plus the new reporter suite).

### Tests (src/core/__tests__/TurnDistributionReporter.test.ts — new file)

9 tests covering:

1. `recordCapExclusion` stores + dedupes per-(agent, round), first ratio wins.
2. Spoken classification for turns > 0.
3. Absent-silent classification (zero turns, no cap/fail data).
4. Absent-capped classification (rounds[] + ratioAtExclusion populated).
5. Absent-failed classification (from failedAgents set).
6. Precedence regression: failedAgents wins over cap exclusion.
7. `finalizeAbsenceEvents` emits exactly one `conversation:agent_absent` per non-spoken entry with correct short status (capped/silent/failed).
8. `finalizeAbsenceEvents` is idempotent across repeated calls.
9. Existing `turn_distribution_updated` + `fairness_alarm` emissions still fire from `report()` (regression guard).

## Deviations from Plan

### [Rule 3 — Blocker] Reporter had no internal state to build a participation report against

**Found during:** Task 1 RED-phase read of `TurnDistributionReporter.ts`.

**Issue:** The plan's `buildParticipationReport` pseudocode referenced `this.getTurnCount(agent)` and assumed the reporter already held per-agent turn counts "where the existing per-agent token share computation reads from." In reality the Phase 13 reporter was stateless — it received `AgentTurnSnapshot[]` per `report()` call and computed shares locally without persisting anything. The plan also assumed a constructor-held EventBus, but the constructor took no arguments and the event bus was passed per-call.

**Fix:** Added internal state to the reporter:
- `private _turnCounts: Map<string, number>` — populated from `report()` snapshots on each call using max-of-previous-vs-current semantics so late-arriving non-monotonic snapshots (e.g., from the forced-rotation branch) cannot regress the count.
- `private eventBus?: EventBus` — set via a new optional constructor parameter; used only by `finalizeAbsenceEvents`. Legacy `new TurnDistributionReporter()` calls still compile.
- `private _capExclusions: Map<string, { rounds: number[]; firstRatio: number }>` + `private _absenceEventsFinalized: boolean` per the plan.

This is a non-breaking extension — the existing `report(snapshots, round, eventBus)` signature is preserved, and all prior `ConversationManager` call sites (`new TurnDistributionReporter()`, `resetForRound()`, `report(...)`) continue to work unmodified.
