    - Test: A failed-then-succeeded chat() call (1 retry) results in 1 logCall with success: false and 1 logCall with success: true
  </behavior>
  <action>
**The bug:** In `LLMProvider.chat()` (lines 61-134), when a call succeeds, `return response` at line 74 triggers the `finally` block (lines 108-129). The `finally` block checks `if (response)` which is truthy on success, so it calls `CostTracker.getInstance().logCall()` a second time with `success: true`. The catch block (lines 75-107) already logs failures. Result: successful calls are logged twice.

**The fix:** Remove the entire `finally` block (lines 108-129). Move the success-path logging into the `try` block, before the `return response` statement. The structure becomes:

```typescript
for (let attempt = 1; attempt <= maxRetries; attempt++) {
  const startTime = Date.now();
  let response: ProviderResponse | undefined;

  try {
    response = await this.performChat(messages, systemPrompt, options);
    // Log successful call ONCE here
    const endTime = Date.now();
    CostTracker.getInstance().logCall({
      provider: this.getProviderName(),
      model: this.getModelName(),
      inputTokens: response.usage?.input_tokens || 0,
      outputTokens: response.usage?.output_tokens || 0,
      cachedReadTokens: (response.usage as any)?.cache_read_input_tokens
        || (response.usage as any)?.prompt_tokens_details?.cached_tokens
        || 0,
      cachedWriteTokens: (response.usage as any)?.cache_creation_input_tokens || 0,
      latency: endTime - startTime,
      success: true,
    });
    return response;
  } catch (error: any) {
    lastError = error;
    const endTime = Date.now();
    // Log failed call (existing code stays as-is)
    CostTracker.getInstance().logCall({
      provider: this.getProviderName(),
      model: this.getModelName(),
      inputTokens: response?.usage?.input_tokens || 0,
      outputTokens: response?.usage?.output_tokens || 0,
      cachedReadTokens: (response?.usage as any)?.cache_read_input_tokens
        || (response?.usage as any)?.prompt_tokens_details?.cached_tokens
        || 0,
      cachedWriteTokens: (response?.usage as any)?.cache_creation_input_tokens || 0,
      latency: endTime - startTime,
      success: false,
    });
    // ... retry logic stays as-is ...
  }
  // NO finally block
}
```

Also remove the now-unused `let success = false;` variable (line 68).
