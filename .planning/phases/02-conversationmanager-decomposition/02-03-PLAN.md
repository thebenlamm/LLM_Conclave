Key ConversationManager judge methods to extract (lines 1394-2050):

`getRoundForEntry(entry)` (line 1394-1414) — Determines which round an entry belongs to by walking the history. Uses judge guidance and compressed entries as round delimiters.

`buildCaseFile()` (line 1416-1508) — Builds a structured text summary of the full discussion organized by round. For each round: lists each agent's position, identifies key points. Used by prepareJudgeContext.

`prepareJudgeContext(judge, discussionText)` (line 1510-1588) — Builds the full judge system prompt. Includes: discussion summary, case file, shallow-agreement warnings, quoting warnings, custom judge instructions (this.judgeInstructions), agent count, round count.

`bestEffortJudgeResult()` (line 1590-1636) — Fallback when judge LLM call fails. Builds a synthesis from the last round's agent responses. Returns { shouldContinue: false, synthesis: "...", ... }.

`judgeEvaluate(judge)` (line 1638-1875) — Main judge evaluation method. Steps:
  1. Cache formatted full discussion
  2. Detect shallow agreement (regex: /I agree|I concur|well said|exactly right|nothing to add|fully support/gi, threshold >= 2)
  3. Detect extensive quoting (current round agents using ">" quotes, threshold > 30% of responses)
  4. Detect rubber-stamp (judge just echoing agents without adding value)
  5. Build discussion text and prepare judge context
  6. Call judge LLM with abort bridging
  7. Parse judge response as JSON
  8. Handle context overflow by compressing and retrying
  9. Fallback to different judge model on failure
  10. Return parsed result with shouldContinue, synthesis, etc.

`conductFinalVote(judge)` (line 1877-2050) — Final vote when max rounds reached without consensus. Builds a ballot with all agent positions, calls judge for final verdict. Returns { winner, reasoning, dissent, ... }.
